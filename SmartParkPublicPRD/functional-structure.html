4. 功能结构设计

4.1 系统功能架构

智慧园区公共服务平台的功能设计采用以用户为中心的思路，从用户视角出发，将功能划分为四个主要模块，确保用户能够直观、高效地使用平台服务。

4.2 核心功能详细设计

4.2.1 首页

功能概述

首页是用户访问平台的第一入口，整合展示园区重要信息和个性化内容，提供直观的服务导航，根据用户角色呈现不同的功能卡片。

详细功能
  1. 园区活动轮播
    • 以轮播图形式展示园区重要活动和新闻
    • 显示活动时间、地点等关键信息
    • 点击跳转至活动详情页
  2. 角色特定内容
    • 企业管理员：展示企业账单、员工管理、企业服务等卡片
    • 企业员工：展示会议室预订、我的申请、常用服务等卡片
    • 公众用户：展示园区简介、入驻指南等卡片
  3. 通知公告
    • 展示最新园区通知和重要公告列表
    • 显示公告标题、发布日期
    • 点击查看公告详情
  4. 政策文件
    • 展示园区最新政策文件列表
    • 显示文件标题、发布日期
    • 点击下载或查看详情

4.2.2 信息公开

功能概述

信息公开模块集中展示园区各类公开信息，包括通知公告、政策文件、企业展示和园区动态，为用户提供全面的园区信息查询服务。

详细功能
  1. 通知公告
    • 公告分类：按类型查看不同类别的通知公告
    • 公告搜索：通过关键词搜索历史公告
    • 公告详情：查看公告完整内容、附件下载
    • 重要公告置顶：突出显示重要通知
    • 已读标记：支持新发布消息有NEW标识，查看后自动移除标识
  2. 政策文件
    • 政策分类：按主题分类展示政策文件
    • 文件搜索：通过关键词搜索相关政策
    • 文件下载：支持下载政策原文
    • 政策解读：提供重要政策的解读说明
    • 已读标记：支持新发布文件有NEW标识，查看后自动移除标识
  3. 园区活动
    • 活动列表：按时间顺序展示园区近期活动
    • 活动详情：查看活动主题、时间、地点、内容介绍
    • 活动报名：支持用户在线报名参加活动
    • 活动提醒：系统自动提醒已报名用户活动时间
    • 活动回顾：活动结束后提供图片、视频等回顾资料
    • 已读标记：新发布活动有NEW标识，查看或报名后移除标识
  4. 需求发布
    • 需求列表：展示园区内企业发布的各类需求
    • 需求详情：查看需求详细描述、联系方式等
    • 需求发布：企业用户可申请发布自身需求
    • 需求筛选：按类型、发布时间等条件筛选需求
    • 需求响应：对感兴趣的需求进行在线响应
  5. 调查问卷
    • 问卷列表：展示园区发布的各类调查问卷
    • 问卷填写：用户可在线填写并提交问卷
    • 结果统计：填写完成后可查看问卷统计结果
    • 已读标记：新发布问卷有NEW标识，填写后自动移除标识
  6. 企业展示
    • 企业名录：展示园区入驻企业列表
    • 企业分类：按行业、规模等维度筛选企业
    • 企业详情：查看企业详细介绍、主营业务
    • 优秀企业展示：重点推荐园区优秀企业案例
  7. 园区动态
    • 园区新闻：园区重要事件、成果的新闻报道
    • 活动回顾：园区举办活动的回顾与总结
    • 图片库：园区活动、场所的图片资源
    • 视频资源：园区宣传、活动的视频资源

4.2.3 服务大厅

功能概述

服务大厅是用户申请各类园区服务的统一入口，按服务类型分类展示，支持搜索、筛选和状态追踪，提供便捷的服务申请体验。

详细功能
  1. 物业服务
    • 事件上报：提交园区事件报告，及时处理紧急情况，含事件类型、事件描述、位置、现场照片
    • 维修申报：申请设备、设施维修服务，支持扫码报修，报修信息包含资产名称、故障描述、现场照片
    • 退租申请：办理企业或个人退租手续，支持修改、提交、撤销功能操作
    • 退园申请：办理企业或个人退园手续，支持修改、提交、撤销功能操作
  2. 配套服务
    • 会议室预订：查看会议室资源，在线预约，系统根据预订时长、企业类型、不同会议室类型、节假日进行费用计算，支持预付款支付、微信支付、银行B2B支付、银行B2C支付
    • 访客门禁申请：为访客申请门禁权限，填写信息包括姓名、手机号、头像，通知到被访人审批
    • 临停缴费：根据车牌查询园区内该车牌的停车费用并缴费，支持所属企业预付款支付、微信支付、银联支付。对于公众访客，支持扫企业停车二维码，形成企业账单后出场，由企业支付账单
    • 停车月卡：购买停车月卡服务，系统根据企业类型进行优惠计算，支持预付款支付、微信支付、银行B2C支付
    • 空调加时：向企业开放其所在空间的空调加时申请权限，企业选择需要加时的空调、按需要加时的时长支付费用，支持预付款支付、微信支付、银行B2B支付、银行B2C支付
    • 电费预充：企业电费预充服务，支持在线缴费（B2B、B2C、微信等支付方式）及查看电费使用情况
    • 加入企业：员工在此录入人脸、身份证号、手机号由企业管理员确认后加入企业并获得门禁权限
    • 企业账单：企业用户可在此进行账单接收、查看、在线缴费与开票，支持多种支付方式（B2B、B2C、微信等），支持插入U盘密钥验证支付操作
    • 开票申请：为已缴费项目申请开具发票，开票内容与用户可查看的企业账单、预订的会议室、空调加时、停车月卡等事项关联
    • 开票变更：修改或增加开票信息，用户可在此进行开票信息增加与变更操作，开票时直接从开票信息中选择抬头
    • 需求发布：发布企业需求，获取园区资源支持，用户可在此进行需求发布申请的新增、删除、查询、修改、提交、撤销
  3. 成长服务
    • 企业注册申请：提供企业注册、变更等相关服务，用户可在此进行企业注册申请的新增、删除、查询、修改、提交、撤销
    • 融资服务申请：为园区企业提供融资对接服务，用户可在此进行融资服务申请的新增、删除、查询、修改、提交、撤销
  4. 增值服务
    • 实验室申请：申请使用园区专业实验室资源，链接至实验室申请系统
    • 普惠算力申请：申请使用园区计算资源服务，链接至普惠算力申请系统
  5. 通用功能
    • 服务搜索：通过关键词快速查找所需服务
    • 服务分类：按类型、热门程度等筛选服务
    • 服务申请表单：根据不同服务提供相应的申请表单
    • 申请进度查询：跟踪服务申请的处理状态
关键流程

物业报修流程

  1. 用户在服务大厅选择维修申报服务
  2. 填写报修类型、位置、问题描述，上传故障照片
  3. 系统自动生成工单并分配给相应的维修人员
  4. 用户可实时查看工单处理进度和处理记录
  5. 维修完成后，系统通知用户确认
  6. 用户确认并评价维修服务，工单结束

电费预充流程

  1. 企业用户在服务大厅选择电费预充服务
  2. 系统显示企业当前电费余额和用电情况
  3. 用户输入需要充值的金额
  4. 选择支付方式（预付款支付、微信支付、银行B2B/B2C支付）完成支付
  5. 系统更新电费账户余额，并提供电子收据
  6. 用户可申请开具电子发票

企业账单支付流程

  1. 企业用户在服务大厅选择企业账单服务
  2. 系统显示所有待缴费的账单列表
  3. 用户选择需要支付的账单，查看明细
  4. 选择支付方式（预付款支付、微信支付、银行B2B/B2C支付）
  5. B2B支付需插入U盘密钥进行验证
  6. 完成支付后，系统更新账单状态
  7. 用户可选择申请开具发票

4.2.4 用户中心

功能概述

用户中心是用户管理个人信息、消息通知、服务记录和相关业务的个人空间，根据用户角色提供不同的功能模块，满足个人和企业管理需求。

详细功能
  1. 个人信息
    • 基本信息维护：修改用户基本信息，如姓名、联系方式等
    • 头像设置：上传、裁剪、设置个人头像
    • 安全设置：修改密码、绑定手机、设置安全问题
    • 登录记录：查看账号登录历史，发现异常登录
  2. 我的消息
    • 消息列表：集中展示用户收到的所有平台消息
    • 消息分类：按未读/已读、消息类型等进行分类
    • 消息详情：查看消息完整内容和相关信息
    • 消息处理：支持标记已读、删除、批量操作
    • 消息设置：设置消息接收方式和提醒频率
  3. 我的预订
    • 会议室预订：查看当前用户的会议室预订列表和详情，可查看会议室临时密码，支持在会议开始前取消或改签
    • 空调加时预订：查看空调加时预订列表和详情，支持在开启前申请取消
  4. 我的活动
    • 活动列表：展示用户已报名参加的园区活动
    • 活动详情：查看活动的时间、地点等详细信息
    • 取消报名：支持在活动开始前取消报名
    • 活动提醒：活动临近时发送提醒通知
  5. 我的申请
    • 申请列表：集中显示用户的所有服务申请记录
    • 申请分类：按申请类型、状态等条件进行筛选
    • 申请详情：查看申请表单内容和处理流程信息
    • 进度追踪：实时显示申请的处理进度和状态
    • 服务评价：对已完成的服务进行评价和反馈
  6. 我的门禁
    • 门禁列表：显示当前用户的所有访客门禁申请列表
    • 门禁详情：查看访客信息和通行时间
    • 通行凭证：生成和查看访客通行二维码
    • 访客门禁审批：对被访人是当前用户的访客申请进行审批
  7. 我的员工
    • 员工列表：企业管理员查看所有企业员工信息
    • 员工详情：查看员工基本信息和使用权限
    • 权限管理：设置和调整员工的系统使用权限
    • 员工审批：处理员工加入企业的申请
    • 员工离职：办理员工离职，撤销相关权限
  8. 我的月卡
    • 月卡列表：显示当前用户购买的停车月卡记录
    • 月卡详情：查看月卡有效期、绑定车牌等信息
    • 月卡续费：支持对即将到期的月卡进行续费
    • 月卡转让：支持将月卡转让给其他用户
  9. 企业管理（企业管理员专属）
    • 企业信息管理：维护企业基本信息、资质文件
    • 员工管理：添加、编辑员工信息，分配权限
    • 部门管理：设置企业部门结构，分配员工
    • 合同管理：查看企业合同，申请续签或变更
  10. 账单与发票（企业管理员专属）
    • 账单查询：查看企业各类费用账单
    • 在线缴费：支持多种支付方式缴纳费用
    • 缴费记录：查询历史缴费记录
    • 发票管理：申请开具发票，查看、下载电子发票
关键流程

企业员工管理流程

  1. 企业管理员登录用户中心，进入企业管理-员工管理
  2. 添加新员工基本信息
  3. 系统自动生成员工初始账号和密码
  4. 员工收到账号激活邮件，完成账号激活
  5. 管理员为员工分配部门和权限
  6. 员工完善个人信息，获得系统访问权限

访客门禁审批流程

  1. 被访人收到系统发送的访客审批通知
  2. 登录用户中心，进入"访客门禁审批"功能
  3. 查看访客信息和来访目的
  4. 选择批准或拒绝访问申请
  5. 系统通知访客审批结果
  6. 批准后，访客获得电子通行证
  7. 访客到达时，被访人收到到达通知

4.4 功能模块关系

平台各功能模块之间存在紧密的业务关联和数据流转关系：

  • 首页与其他模块的关系：首页作为总入口，提供对其他所有模块的快捷访问，同时聚合展示其他模块的核心信息
  • 信息公开与服务大厅的关系：信息公开中的政策可能关联到服务大厅中的具体服务申请
  • 服务大厅与用户中心的关系：用户在服务大厅提交的申请会在用户中心的服务记录中跟踪
  • 用户中心与全平台的关系：用户中心管理的权限和身份信息影响用户在全平台的操作权限

功能模块间的集成原则：

  1. 单点登录：用户只需登录一次，即可访问所有授权功能
  2. 统一消息：所有模块产生的消息统一汇集到消息中心
  3. 数据共享：各模块间的基础数据共享，避免重复录入
  4. 流程衔接：确保跨模块业务流程的无缝衔接

4.5.4 角色视图与模块视图对比分析

按角色组织的功能视图与按模块组织的功能结构各有优势：

对比维度 按模块组织（当前方案） 按角色组织（补充方案）
系统架构清晰度 功能模块边界清晰，系统架构一目了然 功能重叠较多，系统架构边界模糊
用户使用便捷性 需要用户适应系统结构 符合用户思维习惯，易于理解和使用
开发实现难度 功能模块化开发，相对简单 需处理角色间功能交叉和重复逻辑
权限控制精确性 需在每个模块内细分权限 权限与角色直接绑定，控制更精准
功能扩展性 添加新功能模块较容易 新功能需同时考虑多个角色视图的影响
整合建议

综合考虑系统实现和用户体验，建议采用"模块+角色"的混合设计策略：

  • 系统架构采用模块化设计，便于开发和维护
  • 用户界面根据角色动态调整，提供角色化的导航和快捷入口
  • 针对核心业务流程，设计完整的角色化使用路径
  • 系统首页提供角色特定的工作台/控制台视图
  • 权限控制结合角色和具体功能点，实现精细化权限管理
